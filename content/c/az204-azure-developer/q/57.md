    - id: "answer1"
      title: "The data blob"
      correct: true
      explain: "The data blob can be restored because it was soft-deleted in the final step (system error). When blob soft delete is enabled, deleted blobs are retained for the configured retention period and can be restored by calling the Undelete operation. Restoring the base blob also automatically restores all soft-deleted snapshots that were deleted during the soft-delete retention period."
    - id: "answer2"
      title: "Snapshot 1"
      correct: false
      explain: "Snapshot 1 was explicitly deleted before the system error occurred, which soft-deleted it. However, when the data blob and all snapshots were deleted by the system error, Snapshot 1 had already been soft-deleted. When you undelete the base blob, Azure restores snapshots that were deleted during the same retention period, but Snapshot 1 was deleted in a separate operation before the final deletion. While soft delete retains it initially, the documented behavior indicates that explicitly deleted snapshots may not be automatically restored when you undelete the base blob."
    - id: "answer3"
      title: "Snapshot 2"
      correct: true
      explain: "Snapshot 2 can be restored because it was deleted as part of the system error that deleted the data blob. When you call Undelete on the base blob, Azure automatically restores any snapshots that were deleted during the soft-delete retention period along with the blob itself. Since Snapshot 2 was deleted in the same operation as the base blob (the system error), it will be restored when you restore the data blob."
    - id: "answer4"
      title: "None of the resources can be restored"
      correct: false
      explain: "This is incorrect because soft delete is enabled on the storage account. Soft delete protects blobs and their snapshots from accidental deletion by retaining them for a configured retention period. Both the data blob and Snapshot 2 can be successfully restored during the retention period by calling the Undelete operation on the base blob."
