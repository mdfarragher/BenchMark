answers:
    - id: "answer1"
      title: "MongoDB"
      correct: false
      explain: "MongoDB API is designed for document-based data models with rich querying capabilities. While it can store key-value pairs, it's over-engineered for simple key-value scenarios and would incur higher costs than necessary. MongoDB is better suited for complex documents with nested structures and advanced query requirements."
    - id: "answer2"
      title: "Table API"
      correct: true
      explain: "Table API is optimized for key-value and entity storage with a schema-less design. It offers a serverless option for the lowest-cost, best-effort performance tier, and can scale to guaranteed performance with multi-region support for premium tiers. This flexibility matches the requirement for multiple editions at different price points while minimizing Azure costs."
    - id: "answer3"
      title: "Cassandra"
      correct: false
      explain: "Cassandra API is designed for wide-column data models requiring high write throughput and horizontal scaling across multiple regions. While powerful, it's more complex and costly than needed for simple key-value storage. Cassandra excels at time-series data and high-volume writes, not basic key-value operations."
    - id: "answer4"
      title: "Core (SQL)"
      correct: false
      explain: "Core (SQL) API provides rich SQL querying, indexing, and transaction support for document data models. While it supports all Cosmos DB features including multi-region and performance tiers, it's more feature-rich and potentially costlier than Table API for simple key-value scenarios where advanced querying isn't needed."
