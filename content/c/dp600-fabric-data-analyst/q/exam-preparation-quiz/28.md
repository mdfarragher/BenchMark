    - id: "answer1"
      title: "Warehouse"
      correct: true
      explain: "Fabric Warehouse is the correct choice because it fully supports all three requirements: writing data using T-SQL, multi-table transactions with ACID compliance, and security features like dynamic data masking. It's a fully managed relational data warehouse designed for enterprise-grade T-SQL workloads."
    - id: "answer2"
      title: "Semantic Model"
      explain: "A semantic model is used for creating business intelligence reports and analytics, not for storing and managing data with T-SQL operations."
    - id: "answer3"
      title: "KQL Database"
      explain: "A KQL database is designed for log and time-series data analysis using Kusto Query Language, not for T-SQL operations or multi-table transactions."
    - id: "answer4"
      title: "Lakehouse"
      explain: "A lakehouse combines data lake and data warehouse capabilities but does not support multi-table transactions (though it does support single-table transactions)."
